Ruby - Lambda, Proc, and code block

Series: ruby October 21, 2013

Reference: this excellent read by Adam Waxman.

Summary:

  • Lambda and Proc are objects, code block is not (it is a… block of code).

  • Both lambda and Proc are Proc objects:

    (lambda {}).class    # ⇨ Proc 
(Proc.new {}).class  # ⇨ Proc

  • There is at most one block that can appear in a argument list of a function/method:

  • Lambdas check for the number of arguments, Procs don’t.

  • When returning from a lambda, the lambda object reaches its end. When returning from a Proc, the object enclosing the Proc (outer scope) reaches its end.

    def return_from_lambda
  a_lambda = lambda { return 'returned from a lambda' }
  a_lambda.call
  'default return value of the function'
end

def return_from_proc
  a_proc = Proc.new { return 'returned from a Proc' }
  a_proc.call
  'default return value of the function - never reaches'
end

puts return_from_lambda
# ⇨ default return value of the function

puts return_from_proc
# ⇨ returned from a Proc

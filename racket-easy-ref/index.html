3. Syntactic Forms

Local Binding, let, let*, letrec, …

Original link: http://docs.racket-lang.org/reference/let.html

  • With the simplest let form:

    (let ([id val-expr] ...)
  body ...+)

    This:

    • evaluates all val-exprs,
    • create a new location for each id,
    • places the evaluated values into those locations,
    • binds ids to those values lexically,
    • then evaluates body and returns the value of the last expression.

    Something to keep in mind:

    • ids must be distinct.
    • let form is lexically scoped.
    • All bindings are not sequential.

    E.g.

    (let ([x 5]) x)     ;; => 5

(let ([x 5])
  (let ([x 2]
        [y x])
    (list y x)))    ;; => '(5 2)

;; Not working since bindings are not sequential
(let ([x 5]
      [y x])
  (list y x))
;; =>
; x: undefined;
;  cannot reference undefined identifier
; [,bt for context]

  • let proc-id:

    (let proc-id ([id init-expr] ...)
  body ...+)

    This form binds proc-id to a function (λ (id init-expr ...) body ...+) with ids as its arguments that take initial values init-exprs. The binding process works as in let.

    E.g.

    (let fact ([n 10])
  (if (zero? n)
      1
      (* n (fact (sub1 n)))))
;; => 3628800
